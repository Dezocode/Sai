# Decision 0004 — Claude-primary agents use `--suite claude` automation profiles

**Status:** accepted
**Date:** 2026-07-17
**Deciders:** dezocode (via Sai CEO automation and Saul CTO review)
**Supersedes:** none (clarifies Phase 7 of `.ai/INITIALIZE.md`)

## Context

Contractor agent Alpha (`ctr-code-splunk1`) and portfolio agent Mimi
(`claude-code-cli` primary) had Phase 7 profiles generated by
`scripts/agent-automation-spec` without `--suite claude`. The output
documented Cursor Automations UI steps in
`runtimes/claude/automation/profile.md`, which violates runtime truth and
confused principals (CTO REQUEST_CHANGES on PRs #17 and #18).

## Decision

1. **Suite-specific generators:** `scripts/agent-automation-spec` accepts
   `--suite cursor` (default) or `--suite claude`. Claude output documents
   session/scheduled-task protocol only — never Cursor Automations UI steps.

2. **Phase 7 command shape:** Claude-primary agents must pass
   `--suite claude`, `--folder-name <folder-slug>`, and write to
   `.ai/agents/<folder-slug>/runtimes/claude/automation/profile.md`.
   Display names (e.g. "Alpha") must not appear in filesystem paths.

3. **CI enforcement:** `scripts/verify-semantic-hierarchy` rejects
   Claude-primary agents whose `runtimes/claude/automation/profile.md`
   contains Cursor Automations UI markers. `scripts/verify-scaffold-safety`
   regression-tests folder-slug paths in generated specs.

4. **PR triage:** Merge the consolidated init-compliance branch (includes
   PR #19) before rebasing or closing overlapping #17/#18 folder-slug-only
   fixes so CTO P1 is resolved once on `main`.

## Consequences

- New Claude-primary agents cannot complete initialization with invalid
  automation profiles.
- Cursor-primary agents continue using default `--suite cursor` output.
- Fork CI must keep `verify-scaffold-safety` and semantic hierarchy checks
  identical to canonical (see `.ai/shared/references/icm-ci-policy.md`).
